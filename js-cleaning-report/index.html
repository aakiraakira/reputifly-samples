Key rule: Only the Quote Form Submit counts as a PRIMARY conversion for bidding optimization. WhatsApp and call clicks are tracked as SECONDARY (visible in reports but do NOT influence Smart Bidding). This ensures 1 real lead = 1 counted conversion. No more triple-counting.

Recommended Actions

Priority-ordered implementation plan.

1

Fix Conversion Triple-Counting (Immediate)

Keep only ONE conversion action as Primary/Included. Recommend keeping "Lead Form Submission" (Active, cleanest tracking). Set the other two to Secondary and "Not included in account-level goals." This is the single highest-impact fix.

2

Rebuild Landing Page as Lightweight Raw HTML

Replace the Elementor page with a fast, custom-coded landing page. Keep the polished hero section (already done). Add: header with logo, trust section, services breakdown, FAQ, WhatsApp CTA, and footer. Deploy via Reputifly API as raw HTML. Target: under 3 seconds mobile load.

3

Implement Single Clean Conversion Tracking

Use only the Reputifly API form submission as the conversion event. Remove duplicate Google Ads tags, GTM tags, and OCT tags. Implement a unique transaction ID per submission to prevent any recount.

4

Allow 30 Days of Clean Data

After fixing tracking, let the Maximize Conversions algorithm re-learn on real data. Do NOT change bids, budgets, or keywords during this period. Reported conversions will drop to ~1/3 — this is expected and correct.

5

Evaluate Campaign Structure After Clean Data

With 30 days of clean data: if 30+ real conversions/month, switch to Target CPA (set 10-20% above real average CPA). Consider adding broad match versions of top performers. Evaluate if new campaign needed or current can be optimized.
